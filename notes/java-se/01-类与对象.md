## 一、核心概念

### 1. 类和对象的关系

| 概念 | 比喻 | 说明 |
|------|------|------|
| 类（Class） | 图纸 | 定义了属性和行为的模板 |
| 对象（Object） | 真机 | 根据图纸造出来的实体 |

```java
// 类 = 图纸
public class WashingMachine {
    private String brand;
    public void wash() { ... }
}

// 对象 = 造出来的真机
WashingMachine wm = new WashingMachine("海尔", 10);
```

### 2. `new` 关键字做了什么？

- 在**堆内存**中分配空间
- 初始化成员变量（默认值）
- 调用构造方法
- 返回对象的引用（存的是地址）

### 3. `private` 封装

```java
private String name;  // 外部不能直接 obj.name访问
```

**为什么封装？** 保护数据，只能通过方法访问，可以加校验逻辑。面试必问！

### 4. `this` 关键字

`this` 代表**当前对象**的引用，最常用于：
- 区分成员变量和局部变量同名：`this.name = name;`
- 调用本类的其他构造方法：`this()` 或 `this(参数)`

### 5. 构造方法

```java
public Student(String name, int score) {  // 没有返回值类型，连void都没有
    this.name = name;
    this.score = score;
}
```

- 方法名 = 类名
- **没有返回值类型**（不是 void）
- `new` 的时候自动调用
- 不写的话，Java 默认给一个无参构造

## 三、面试考点速记

| 考点 | 一句话答案 |
|------|-----------|
| 类和对象区别 | 类是模板/图纸，对象是实例/真机 |
| `new` 做了什么 | 堆内存分配 → 初始化 → 调构造 → 返回引用 |
| `this` 是什么 | 当前对象的引用，区分同名字段 |
| `private` 作用 | 封装，外部不能直接访问，通过 getter/setter |
| 构造方法特点 | 和类同名、无返回值类型、new时自动调用 |
