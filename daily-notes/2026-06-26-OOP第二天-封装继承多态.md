## 二、今天主攻：封装 → 继承 → 多态

### 1️⃣ 封装（Encapsulation）完善 — getter/setter（30min）

**为什么需要 getter/setter？**
- `private` 封了字段，外部读不到也改不了 → 需要"门禁"
- getter 控制**读**的权限，setter 控制**写**的权限
- 可以在 setter 里加校验逻辑（比如 age 不能 < 0）

**标准写法：**
```java
public class Person {
    private String name;
    private int age;

    // getter：返回字段值
    public String getName() {
        return name;
    }

    // setter：设置字段值（可以加校验）
    public void setAge(int age) {
        if (age < 0) {
            throw new IllegalArgumentException("年龄不能为负数");
        }
        this.age = age;
    }
}
```

**权限修饰符速记表：**

| 修饰符 | 本类 | 同包 | 子类 | 任何地方 |
|--------|------|------|------|---------|
| private | ✅ | ❌ | ❌ | ❌ |
| default(不写) | ✅ | ✅ | ❌ | ❌ |
| protected | ✅ | ✅ | ✅ | ❌ |
| public | ✅ | ✅ | ✅ | ✅ |

### 2️⃣ 继承（Inheritance）— extends（30min）⭐

**一句话：** 子类拿到父类的字段和方法，不用重写一遍

```java
public class Animal {
    private String name;

    public void eat() {
        System.out.println("吃东西");
    }
}

public class Dog extends Animal {  // Dog 继承 Animal
    public void bark() {
        System.out.println("汪汪叫");
    }
}
// Dog 同时有 eat() 和 bark()
```

**super 关键字两种用法：**

| 用法 | 写法 | 什么时候用 |
|------|------|-----------|
| 调父类构造 | `super(参数)` | 子类构造方法里，**必须第一行** |
| 调父类方法 | `super.方法名()` | 子类重写后还想用父类原来的逻辑 |

```java
public class Dog extends Animal {
    public Dog(String name) {
        super(name);  // 调父类构造，必须第一行
    }

    @Override
    public void eat() {
        super.eat();  // 先调父类的 eat
        System.out.println("吃骨头");  // 再加自己的
    }
}
```

**方法重写 @Override vs 重载：**

| | 重写 Override | 重载 Overload |
|------|-------------|-------------|
| 发生位置 | 父子类之间 | 同一个类里 |
| 方法签名 | **完全相同** | 参数列表不同 |
| 注解 | @Override | 无 |
| 运行时 | 多态体现 | 编译时决定 |

**final 关键字：**
- `final class` → 类不能被继承（如 String）
- `final 方法` → 方法不能被重写
- `final 变量` → 变成常量，不能改

### 3️⃣ 多态（Polymorphism）— 面试最爱问（30min）⭐⭐

**一句话：** 父类引用指向子类对象，同一个方法不同表现

```java
Animal a = new Dog();  // 父类引用 → 子类对象
a.eat();               // 调的是 Dog 的 eat()，不是 Animal 的
```

**核心规则：**
- **编译看左边，运行看右边** — 左边 Animal 决定能调哪些方法，右边 Dog 决定实际跑哪个
- 多态的前提：必须有继承 + 方法重写

**向上转型（自动）：**
```java
Animal a = new Dog();  // 子→父，自动，安全
```

**向下转型（强制，危险⚠️）：**
```java
Animal a = new Dog();
Dog d = (Dog) a;       // 父→子，强制转换，必须原本就是 Dog
```

**instanceof 安全检查：**
```java
if (a instanceof Dog) {
    Dog d = (Dog) a;
    d.bark();
}
```
